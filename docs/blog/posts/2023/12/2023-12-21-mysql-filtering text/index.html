MySQL Fitering Text

MySQL DB에서 text 데이터를 필터링하는 명령을 정리합니다. 데이터는 기존에 생성 한 아래의 test_table 데이터를 사용합니다.

mysql> SELECT * FROM test_table;
+----+-------------+---------------+
| id | name        | phone         |
+----+-------------+---------------+
|  1 | tony        | 010-1111-1112 |
|  3 | Adel Mon    | 010-2232-1112 |
|  4 | john kim    | 010-2222-2222 |
|  5 | john junior | 010-1231-2224 |
+----+-------------+---------------+
4 rows in set (0.00 sec)

Like

Text가 저장된 필드의 특정한 패턴을 찾아낼 때 LIKE를 사용합니다.

% 패턴

LIKE와 같이 사용되는 패턴 검색에 사용되는 %는 없거나 하나이상의 문자열을 표현합니다.

mysql> SELECT * FROM test_table WHERE name LIKE 'john%';
+----+-------------+---------------+
| id | name        | phone         |
+----+-------------+---------------+
|  4 | john kim    | 010-2222-2222 |
|  5 | john junior | 010-1231-2224 |
+----+-------------+---------------+
2 rows in set (0.00 sec)

검색 결과로 john 또는 john이후에 다른 문자열이 있는 레코드가 추출됩니다.

_ 패턴

LIKE와 같이 사용되는 패턴 검색에 사용되는 _는 문자 하나를 표현합니다.

mysql> SELECT name FROM test_table WHERE name LIKE 'ton_';
+------+
| name |
+------+
| tony |
+------+
1 row in set (0.00 sec)

ton으로 시작하는 문자에서 임의의 문자 하나가 추가된 Text를 필터링합니다. 결과는 총 4개 문자를 갖습니다.

NOT LIKE

필터링 시 제외해야하는 문자를 설정합니다. 이 작업은 대소문자를 구분하기 때문에 입력 문자열에 주의해야 합니다.

mysql> SELECT name FROM test_table WHERE name NOT LIKE 'to%';
+-------------+
| name        |
+-------------+
| Adel Mon    |
| john kim    |
| john junior |
+-------------+
3 rows in set (0.00 sec)

tony를 갖는 레코드만 제외되었습니다.

Wildcard Position

특정 위치의 문자를 검색하는 예제를 정리합니다.

mysql> SELECT name FROM test_table
    -> WHERE name LIKE '%r';
+-------------+
| name        |
+-------------+
| john junior |
+-------------+
1 row in set (0.00 sec)

마지막이 r로 끝나는 패턴을 찾습니다.

mysql> SELECT name FROM test_table WHERE name LIKE '___l%';
+----------+
| name     |
+----------+
| Adel Mon |
+----------+
1 row in set (0.01 sec)

세 번째 문자가 ㅣ인 레코드를 찾습니다.

WHERE, IN

IN을 이용하여 OR조건을 간단하게 표현할 수 있습니다.

mysql> SELECT name, phone FROM test_table WHERE phone IN ('010-1111-1112', '010-2232-1112');
+----------+---------------+
| name     | phone         |
+----------+---------------+
| tony     | 010-1111-1112 |
| Adel Mon | 010-2232-1112 |
+----------+---------------+
2 rows in set (0.00 sec)

전화번호 2개에 대한 OR고전을 IN으로 변경하여 레코드를 추출합니다.

BETWEEN

두 데이터 사이에 존재하는 값을 필터링합니다. BETWEEN을 연습하기 위해 숫자값을 갖는 age 컬럼을 생성합니다.

mysql> ALTER TABLE test_table ADD COLUMN age INT;
Query OK, 0 rows affected (0.58 sec)
Records: 0  Duplicates: 0  Warnings: 0

age컬럼을 INT 형식으로 생성한 하고 결과를 확인합니다.

mysql> SELECT * FROM test_table;
+----+-------------+---------------+------+
| id | name        | phone         | age  |
+----+-------------+---------------+------+
|  1 | tony        | 010-1111-1112 | NULL |
|  3 | Adel Mon    | 010-2232-1112 | NULL |
|  4 | john kim    | 010-2222-2222 | NULL |
|  5 | john junior | 010-1231-2224 | NULL |
+----+-------------+---------------+------+
4 rows in set (0.00 sec)

default값을 지정하기 않았기 때문에 NULL로 표시됩니다. age정보를 입력합니다.

mysql> UPDATE test_table SET age = 10 WHERE id = 1;

입력된 age 컬럼 정보를 이용해서 BETWEEN 명령을 수행합니다.

mysql> SELECT * FROM test_table;
+----+-------------+---------------+------+
| id | name        | phone         | age  |
+----+-------------+---------------+------+
|  1 | tony        | 010-1111-1112 |   10 |
|  3 | Adel Mon    | 010-2232-1112 |   20 |
|  4 | john kim    | 010-2222-2222 |   15 |
|  5 | john junior | 010-1231-2224 |   20 |
+----+-------------+---------------+------+
4 rows in set (0.00 sec)

mysql> SELECT * FROM test_table WHERE age BETWEEN 15 and 20;
+----+-------------+---------------+------+
| id | name        | phone         | age  |
+----+-------------+---------------+------+
|  3 | Adel Mon    | 010-2232-1112 |   20 |
|  4 | john kim    | 010-2222-2222 |   15 |
|  5 | john junior | 010-1231-2224 |   20 |
+----+-------------+---------------+------+

age가 15 ~ 20사이인 레코드가 필터링 됩니다.
